# stv:clarify: Context Brief 생성

유저의 요구가 불명확할 때 **한 번에 하나씩** 질문하여 Context Brief를 만든다.
명확해지면 즉시 다음 단계로 넘어간다.

---

## 1. 불명확 판단 기준

다음 중 하나라도 해당하면 이 스킬을 트리거:

- 목표가 여러 가지로 해석 가능
- 범위(In/Out)가 특정되지 않음
- 성공 기준이 없거나 모호
- 기술적 제약 조건 불명

## 2. 질문 규칙

- **한 번에 하나만** 질문한다 (묶어서 던지지 마라)
- 질문 전에 **추론 가능한 것은 추론해서 제시**하고 확인만 받아라
- 3회 질문 내에 Brief를 완성하지 못하면, 현재까지의 Brief를 제시하고 유저 확인을 받아라
- 이미 명확한 요청에는 이 스킬을 발동하지 마라

## 3. Context Brief 포맷

질문이 완료되면 아래 포맷으로 정리:

```markdown
## Context Brief: [Task Title]

**Goal**: 무엇을 달성해야 하는가 (1-2문장)

**Scope**:
- In: 포함할 것
- Out: 명시적으로 제외할 것

**Success Criteria**: 완료 판단 기준 (측정 가능하게)

**Constraints**: 기술적/시간적/조직적 제약 (있을 경우만)
```

## 4. Brief 완성 후

- Brief를 유저에게 보여주고 확인받는다
- 확인되면 즉시 다음 단계(구현, 계획, stv:new-task 등)로 넘어간다
- Brief는 이후 작업의 SSOT가 된다
